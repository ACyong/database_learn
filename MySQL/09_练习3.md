## 练习3：

1. order by：
```
1、将英雄按防御值从低到高排序
select * from users order by gongji asc;

2、将蜀国英雄按攻击值从高到低排序
select * from users where country = "蜀国" order by gongji desc;

3、将魏蜀两国的男英雄中名字为三个字的英雄按防御值升序排列
select * from users where country in ("蜀国","魏国") and sex="M" (and name like "___") order by fangyu asc;
```

2. limit：
```
1、查找攻击值前三名且名字不为空值的蜀国的英雄的姓名、攻击值及国家
select name,gongji,country from users where country="蜀国" and (name is not null) order by gongji desc limit 3;

2、查找防御值倒数第二名至倒数第四名的蜀国英雄的记录
select * from users where country="蜀国" order by fangyu limit 1,3;
```

3. group by：
```
1、查询 users 表中一共有几个国家
select country from users group by country;

2、计算所有国家的平均攻击力
select country,avg(gongji) from users group by country;

3、查找所有国家中 英雄数量最多 的前2名 的国家名称及英雄数量
select country,count(*) as number from users group by country order by number limit 2;
```

4. having：
```
1、找出平均攻击力大于105的国家的前2名,显示国家名和平均攻击力
select country,avg(gongji) as pjgj from users group by country having pjgj>150 order by pjgj desc limit 2;
```

5. distinct：
```
1、users 表中一共有多少个国家
select distinct country from users;

2、计算蜀国一共有多少个英雄
select count(distinct name) from users where country="蜀国";
```

注意:
```
1、distinct处理的是distinct与from之间的所有字段，所有字段必须全部相同才能去重
2、distinct不能对任何字段做聚合处理
```

6. 运算符：
```
1、查询显示时所有英雄攻击力全部 * 10
select id,name,country,gongji*10 as xgj from users;

2、查询显示时所有英雄的防御力 + 5
select id,name,country,fangyu+5 as xfy from users;
```

7. 聚合函数：
```
1、攻击力最强值是多少
select max(gongji) as best from users;

2、统计一下表中id,name字段分别有多少条记录
select count(id),count(name) from users;
注：空值NULL被统计，空字符串""不会被统计

3、计算蜀国英雄的总攻击力
select sum(gongji) from users where country = "蜀国";

4、统计蜀国英雄中攻击值大于200的英雄的数量
select count(*) from users where country="蜀国" and gongji>200;
```

8. 删除所有表记录
```
DELETE FROM users;
```

9. 删除表
```
DROP TABLE users;
```

10. 删除库 users
```
DROP DATABASE users;
```
